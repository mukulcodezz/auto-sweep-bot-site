EVIDENTIARY BUILD LOG

Build History & Security Architecture

This changelog documents the structural evolution of the Auto-Sweep Bot. Each phase is gated by strict engineering priorities ensuring zero-trust integrity.

PHASE 5: IN PROGRESS

Mainnet-ready Signer Interface

A generic remote-signing interface is built. The wallet key never enters this process, and signing requests go to whatever custody backend answers the contract (Turnkey, Fireblocks, a self-hosted HSM bridge). Not yet wired to a live provider, so mainnet stays locked.

PHASE 4: DISCLOSURE

Independent Security Review

FIXED CWE-287

Not a marketing badge, but an evidentiary report. The external review identified a race condition in the sweep queue. Fixed via a semaphore-lock on the transaction buffer.

PHASE 3: ISOLATION

Splitter & Read-Only Safety

Implementation of "Read-Only Splitter" for group environments. Commands issued from non-admin chat IDs are restricted to information queries only.

PHASE 2: INTEGRATION

Jupiter Protocol Auto-Sweep

Integrated Jupiter's aggregator for best-price swaps, with two independent safety ceilings: a configurable slippage tolerance (1% default) that Jupiter enforces on-chain, and a 5% price-impact ceiling that rejects a thin route before it's ever signed.

PHASE 1: FOUNDATION

Auth & Vault Core

Why Auth came first: No feature logic was written until the encryption layer was validated. The system uses AES-256-GCM for all local storage, indexed by a hardware-bound key.
