◇ Reference / command surface

Command reference.

Every command runs in your 2-person group chat. Nothing works until your numeric Telegram ID is on the allowlist — there's no in-bot signup, the operator adds your ID in config before the bot runs.

01

Access

Access is a fixed allowlist set in config before the bot ever runs. There's no in-bot signup or request-access command — your ID has to be added by whoever operates the bot. A non-allowlisted user gets no response in the group and a flat "Not authorized." in a DM. Every attempt, allowed or denied, lands in the audit log.

Obtain your numeric ID · separate public bot, not a bot command

@userinfobot
// Message this on Telegram. Give the operator the numeric ID it returns — not your @username — to be added to ALLOWLIST_TELEGRAM_IDS.
02

Commands

the whole surface

Four commands, all usable in the group chat. None of them signs a transaction — the only thing that moves funds is the automatic sweep + cold-forward, which needs no command. Typo one and the bot suggests the closest real command.

Confirm you're seen
$ /start

Confirms you're allowlisted and shows whether auto-sweep is ON or OFF.

Your identity
$ /whoami

Returns your Telegram ID and username.

Command list
$ /help

Lists every command and how sweep behaves.

Toggle + wallet status
$ /autosweep on|off|status

status returns wallet address, live SOL balance, and cold-forward config. on/off toggles auto-swap; the setting persists in data/settings.json.

03

Sweep behavior

automatic · no command

Deposits are always announced. Swapping only happens when auto-sweep is ON.

  • inA SOL or SPL-token deposit is detected by the watchers and announced in the group with amount, sender, and a tx link.
  • swapIf sweep is ON, a non-SOL token is routed to SOL via Jupiter — Raydium as fallback — subject to a slippage ceiling and a minimum-output floor. A route that doesn't clear the floor is skipped, not forced.
  • outSwept SOL above the hot-wallet buffer forwards to the cold WITHDRAW_ADDRESS if one is set. The emptied token account is closed and its rent reclaimed.
04

Upcoming

planned · not in this build

Designed but not shipped. None of these commands exist in the running bot yet.

Planned
$ /login <username> <password>

Argon2 password login opening a 30-minute session. Today access is the numeric-ID allowlist only — there is no password step.

Planned
$ /sweep <otp>

Force an immediate full pass over the wallet, gated by a burn-on-use TOTP. Sweeping already runs automatically today; the manual trigger and OTP layer aren't built.

Planned
$ /send <otp> send 1.5 to alice, 0.2 to bob

Atomic multi-recipient splitter resolving aliases from a contacts file or a literal address. Not wired to the bot yet.

Planned
$ single-use TOTP + group secret scrub

Fresh 6-digit code per fund-moving command, burned on use, plus deletion of any secret typed into a group by mistake. None of this is active in the current build.
