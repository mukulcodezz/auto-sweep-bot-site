◇ Reference / command surface

Command reference.

Read-only commands work anywhere. Anything that moves funds is DM-only and needs a fresh one-time code. Nothing below works until your numeric ID is on the allowlist.

Access & login Read-only Fund-moving Error codes
01

Access & login

Access is a fixed allowlist set in config before the bot ever runs. There's no in-bot signup or request-access command — your ID has to be added by whoever operates the bot.

Obtain your numeric ID · separate public bot, not a bot command

@userinfobot
// Message this on Telegram. Send the numeric ID it returns — not your @username.

Admin login · DM only

$ /login <username> <password>
// Opens a 30-minute admin session. Password is Argon2-hashed — never stored or logged in plaintext. /logout ends it early.
02

Read-only

safe in groups

No on-chain state changes. These never move money.

Recognition check
$ /whoami

Returns your Telegram ID and current session status.

Wallet status · needs a session
$ /status

Returns the wallet address and live SOL balance. Read-only.

/help, /start and /whoami work with no login. Typo a command and the bot suggests the closest real one.

03

Fund-moving

restricted · DM only
Strict DM protocol

These are ignored in group chats. If a password or OTP is typed into a group by mistake, the bot deletes that message (where it has permission) and burns the OTP immediately so it can't be reused. No account freeze — an immediate scrub.

Force an immediate sweep

$ /sweep <otp>
// Incoming tokens are already swept in the background with no command. This forces an immediate pass over the whole wallet.

Split-send syntax

$ /send <otp> send [amount] to [alias|address], …

// example
$ /send 982103 send 1.5 to alice, 0.2 to bob
  • otp6-digit authenticator code. Required fresh even mid-session, burned after one use — the same code can never be replayed.
  • aliasA name from the admin-maintained contacts file, or a literal wallet address typed directly. The bot never invents a destination from free text.
