Set up your own instance

This isn't a hosted service. You run it yourself, locally, on your own machine. Every step below happens in a terminal and a .env file, not a web form. No key ever leaves your machine.

01

Allowlist your Telegram ID

REQUIRED

Message @userinfobot on Telegram to get your numeric ID (not your @username, usernames can change). Add it to ALLOWLIST_TELEGRAM_IDS in your .env file before starting the bot. This is the hard access gate. Nothing else works without it.

ALLOWLIST_TELEGRAM_IDS=540291,882731
02

Admin credentials

Pick a username and password, then hash the password locally. The plaintext never touches a config file.

npx tsx scripts/hash-password.ts "your-password"
info PASTE THE OUTPUT INTO ADMIN_PASSWORD_HASH IN .ENV
03

Generate a TOTP secret

Every fund-moving command needs a fresh code from this, even mid-session. Generate it locally and enter it manually into your authenticator app (Google Authenticator, 1Password, etc.).

npx tsx scripts/gen-totp-secret.ts
PASTE THE PRINTED SECRET INTO ADMIN_TOTP_SECRET, THEN ADD IT TO YOUR AUTHENTICATOR APP AS A MANUAL/BASE32 ENTRY
OZKUAUSEFQKSEAKF content_copy
04

Generate the wallet keystore

Set KEYSTORE_PASSWORD (8+ characters) in .env, then generate a fresh Solana keypair. It's written straight to an AES-256-GCM encrypted file on your own disk. There is no upload step, no paste-your-private-key field, anywhere in this process.

npx tsx scripts/init-keystore.ts

PRINTS A PUBLIC KEY. FUND IT FROM A DEVNET FAUCET, NOT REAL MONEY

05

Start the bot and test it

READY

Run it locally, then DM the bot on Telegram: /whoami to confirm you're recognized, /login, then /status to see the funded devnet wallet.

npm run dev
