1. Auto-Sweep Engine

ULTRA_LOW_LATENCY

Unlike a bot that polls the RPC on an interval, Auto-Sweep subscribes directly to the SPL Token program, filtered to accounts owned by the wallet. Any balance change notifies immediately, no polling lag.

CODE_SNIPPET COPY
connection.onProgramAccountChange( TOKEN_PROGRAM_ID, (info) => { const account = AccountLayout.decode(info.accountInfo.data); if (account.amount > 0n) sweepMint(account.mint, account.amount); }, { commitment: "confirmed", filters: [{ dataSize: 165 }, { memcmp: { offset: 32, bytes: owner } }], } );
  • check_circle Notification-driven, not poll-driven: no fixed interval to wait out.
  • check_circle "Confirmed" commitment before acting, balanced against reorg risk, not just raw speed.

2. Routing & Guardrails

JUP_AG_AGGREGATOR

All swaps route through Jupiter's aggregator for the best available price, with two independent safety ceilings so a thin or bad route can't quietly bleed value even if it's technically "within slippage."

Slippage Tolerance 1% (configurable)
Price Impact Ceiling REJECT > 5%

If price impact exceeds 5%, the route is rejected before it's ever signed, not signed-with-a-warning, rejected outright.
