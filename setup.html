Get started.

Node, a Telegram bot, and a devnet faucet. The bot prints a public key at the end — you fund that from a faucet.

Install

$ npm install
01

Copy the env file

Everything below fills into .env.

$ cp .env.example .env
02

Bot token, group chat & allowlist

TELEGRAM_BOT_TOKEN from @BotFather. TELEGRAM_GROUP_CHAT_ID = your 2-person group's ID (a negative number from @getidsbot). ALLOWLIST_TELEGRAM_IDS = your and your partner's numeric IDs from @userinfobot, comma-separated. This is the hard access gate — nothing works without it.

03

Create the wallet

Leave SIGNER_TYPE=local, set KEYSTORE_PASSWORD (8+ chars), then generate. It prints a public key — fund it from a devnet faucet, never real money.

$ npx tsx scripts/init-keystore.ts

Already have a key? scripts/import-keystore.ts imports it; scripts/export-keystore.ts exports for Phantom.

04

Optional — cold-forward & tuning

Set WITHDRAW_ADDRESS to auto-forward swept SOL above HOT_WALLET_BUFFER_LAMPORTS to a cold wallet. Tune SWEEP_SLIPPAGE_BPS, and set AUTOSWEEP_ENABLED to start with sweeping on. A dedicated SOLANA_RPC_URL + SOLANA_WS_URL gives reliable 24/7 websocket watching.

Run it

$ npm run dev

In your group chat, send /start to confirm you're allowlisted, then /autosweep on, and send a token to the wallet to watch it sweep. For a VPS, npm run start:prod builds and runs it under PM2.

Full command list on the command reference. The mainnet guard is not optional — the bot won't start against mainnet-beta with a local keystore.
