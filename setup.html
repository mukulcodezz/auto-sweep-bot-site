Get started.

Node, an authenticator app, and a devnet faucet. The bot prints a public key at the end — you fund that from a faucet.

Install

$ npm install
01

Copy the env file

Everything below fills into .env.

$ cp .env.example .env
02

Bot token & allowlist

TELEGRAM_BOT_TOKEN from @BotFather. ALLOWLIST_TELEGRAM_IDS = your numeric ID from @userinfobot, comma-separated for multiple admins. This is the hard access gate — nothing works without it. ADMIN_USERNAME — pick anything; it's checked with the password at /login.

03

Hash your password

Paste the output into ADMIN_PASSWORD_HASH. The plaintext is never stored.

$ npx tsx scripts/hash-password.ts "your-password"
04

Generate a TOTP secret

Paste into ADMIN_TOTP_SECRET and add it to an authenticator app. Fund-moving commands ask for the 6-digit code fresh, every time, even mid-session — burned on use.

$ npx tsx scripts/gen-totp-secret.ts
05

Create the wallet

Leave SIGNER_TYPE=local, set KEYSTORE_PASSWORD (8+ chars), then generate. It prints a public key — fund it from a devnet faucet.

$ npx tsx scripts/init-keystore.ts
06

Optional — contacts & looser parsing

Copy data/contacts.example.json to data/contacts.json for /send aliases — it can only pay names listed here or a literal address, never one invented from free text. Set GROQ_API_KEY to parse looser phrasing; without it a deterministic regex handles "<amount> to <name>".

Run it

$ npm run dev

In Telegram, DM the bot /help. Then /login, /status, and send a token to the wallet to watch it sweep.

Full command list on the command reference. The mainnet guard is not optional — the bot won't start against mainnet-beta with a local keystore.
