Security Integrity Model

fingerprint

Gated by Numeric ID

Commands only execute for pre-registered Telegram IDs. Username spoofing via display name change is technically impossible within our logic layer.

timer_10_alt_1

OTP Burn-on-Use

Every sensitive transaction requires an out-of-band Time-based One-Time Password. Codes are invalidated immediately upon first signature attempt.

history_edu

Tamper-Evident Audit Log

Every command and server response is SHA-256 hashed and appended to a read-only log accessible via the secure console. No history deletion permitted.

security_update_good

Isolated Signer Boundary

Private keys never touch the Telegram API server. Transactions are built in a secure enclave and passed as raw-tx for the bot to broadcast.
