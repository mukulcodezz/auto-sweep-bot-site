Command Reference

Technical specifications for the Auto-Sweep Bot CLI. All commands are strictly typed and require cryptographic validation for fund-moving operations.

PHASE 01

Setup & Allowlisting

Access is a fixed allowlist of Telegram accounts, set in the bot's configuration before it ever runs. There's no in-bot signup or request-access command. Your ID has to be added by whoever runs the bot before anything below works.

OBTAIN YOUR NUMERIC ID (not a bot command, a separate public bot)
@userinfobot
// Message this on Telegram. Send the numeric ID it returns, not your @username, to whoever manages the allowlist.
ADMIN LOGIN (DM ONLY)
/login <username> <password>
// Starts a 30-minute admin session. Password is argon2-hashed, never stored or logged in plaintext. /logout ends it early.
PHASE 02

Read-only Commands

These commands do not trigger state changes on-chain and are safe for use in group contexts.

RECOGNITION CHECK
/whoami
Response: your Telegram ID + current session status
WALLET STATUS (needs an active session)
/status
Response: wallet address + live SOL balance, read-only
RESTRICTED

Fund-Moving Operations

warning

STRICT DM PROTOCOL: The following commands are ignored if called from a group chat. If a password or OTP gets typed into a group by mistake, the bot deletes that message (if it has permission) and burns the OTP immediately so it can't be reused. There's no account freeze, just an immediate scrub.

FORCE AN IMMEDIATE SWEEP
/sweep <otp>
// Incoming tokens are already swept automatically in the background with no command. This just forces an immediate pass over the whole wallet.
SPLIT-SEND SYNTAX
/send <otp> send [amount] to [alias/address], ...
EXAMPLE EXECUTION
/send 982103 send 1.5 to alice, 0.2 to bob
otp: 6-digit code from your authenticator app, required fresh even mid-session, burned after one use, so the same code can never be replayed.
alias: A name from the admin-maintained contacts list, or a literal wallet address typed directly. The bot can never invent a payout destination from free text.

Security & Error Handling

ERROR_CODE DESCRIPTION SYSTEM_RESPONSE
ERR_UNK_CONTACT Recipient isn't a known contact or a valid address. TX_REJECTED
ERR_OTP_INVALID Code doesn't match the current TOTP window, or is missing entirely. AUTH_FAIL_RETRY
ERR_OTP_REPLAY Code was already used once: burned on first use, can't be replayed. AUTH_FAIL_RETRY
ERR_SLIPPAGE Swap would exceed the configured slippage ceiling (default 1%). HALT_SWAP
ERR_IMPACT_H Route's price impact exceeds the 5% ceiling, rejected before it's ever signed. HALT_SWAP
ERR_SECRET_IN_GROUP Password or OTP typed outside a DM. MSG_SCRUBBED + OTP_BURNED
