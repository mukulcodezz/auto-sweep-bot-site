01 / CAPABILITIES

Three moving parts. One discipline.

Auto-sweep

Tokens in, SOL out

Watches the wallet and its child token accounts. Any SPL token that arrives is routed to SOL through Jupiter automatically — you don't send a command, and you don't get left holding a long tail of dust.

Best-rate routing

Gated on liquidity

Slippage and a separate price-impact ceiling both apply. A route that would exit into an illiquid pool is rejected before it is ever signed — not unwound afterward.

Cold forward

Hot wallet stays thin

Swept SOL — and direct SOL deposits — forward to a cold wallet above a set buffer, so the Telegram-operated hot wallet never sits on more than it needs. Emptied token accounts are closed and their rent reclaimed.

02 / INTEGRITY MODEL

Why this isn't another drainer bot.

Gated by numeric ID

Not by @username

Commands execute only for pre-registered Telegram user IDs. Usernames change hands; numeric IDs don't. Spoofing by renaming a display name does nothing here.

Mainnet-refuse guard

Won't run real funds on a hot key

The config loader hard-refuses to start against mainnet-beta with a local keystore signer. It's a guard clause in code, not a policy line — mainnet needs a custody signer wired in first.

Isolated signer

The key never reaches Telegram

Parsing and chat handling never touch key material. Signing lives behind one Signer interface — local encrypted keystore today, KMS or custody for mainnet.

Tamper-evident log

Break one line, break the chain

Every attempt is appended to a hash-chained log. Editing any past entry breaks verification for every line after it — tampering becomes detectable, not silent.

05 / UPCOMING

On the roadmap — not live yet.

These are designed but not shipped. Nothing below runs in the current build; the bot today is auto-sweep, cold-forward, announcements, and the allowlist gate.

Planned

Atomic payout splitter

/send 1 to alice, 2 to bob paying multiple recipients in one transaction, resolving aliases from a fixed contacts file or a literal address. Optional Groq parsing for looser phrasing.

Planned

Admin login + single-use TOTP

/login with an Argon2 password and a 30-minute session, plus a burn-on-use TOTP required fresh for every fund-moving command. Today access is the numeric-ID allowlist only.

Locked

Mainnet custody

A KMS / HSM-backed signer behind the existing Signer interface, with withdrawal caps and command rate limits, plus an independent third-party audit before real funds.
